# PPTX Skill

Presentations are layout-heavy and slide-bounded. Unlike PDFs, content on a slide can't flow to the next page — if text overflows, it just disappears off the bottom. Verification must confirm content fits each slide.

## Renderer choice

**Primary: python-pptx.** Only serious tool for generating .pptx programmatically. Gives you full control over slide layouts, shapes, text frames, images, tables.
```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation()  # or Presentation('template.pptx') to inherit theme
slide = prs.slides.add_slide(prs.slide_layouts[1])  # Title + Content layout
slide.shapes.title.text = "Slide Title"
slide.placeholders[1].text = "Body bullet 1"
prs.save('output.pptx')
```

**Secondary: pandoc markdown → pptx.** Quick for content-only decks without custom styling. Limited control.
```bash
pandoc deck.md -o output.pptx --reference-doc=template.pptx
```

## Workflow

1. **Ask for a template if branded.** If the user or their organization has a corporate template (colors, fonts, logo on every slide), use `Presentation('template.pptx')` — the agent should NOT invent brand identity.
2. **Sketch structure first.** What slides? What goes on each? Block out content before writing code. 10 slides × 1 title + 3-5 bullets is a common shape for pitches.
3. **Generate.**
4. **Verify.** PPTX isn't directly viewable — `read_file` can't render it. Two paths:
   - **Visual** (preferred): convert to PDF and inspect. `libreoffice --headless --convert-to pdf output.pptx` → `read_file output.pdf` → check each slide.
   - **Structural** (fast): re-open with python-pptx and walk slides, checking that each has a title, expected number of bullets, and text doesn't obviously overflow (text frame height vs font size).
5. **Fix and re-run.** Cap at 3 iterations. Stop on objective bugs, don't chase subjective polish.

## What to verify in the PDF render

- Every slide has a visible title
- No text clipped at the bottom (sign of overflow)
- Images don't cover text or overflow the slide
- Body text legible at projector scale — 18-24pt body, 28-40pt titles
- No empty slides (generated by accident)
- Consistent visual style across slides (no random color or font jumps)

## Layout discipline

Three rules that catch 90% of bad decks:

1. **One idea per slide.** If a slide has more than 6 bullets, it's two slides.
2. **Titles are sentences, not labels.** "Q3 revenue up 23%" beats "Q3 Revenue."
3. **Leave whitespace.** Margins matter — don't fill every pixel.

## Using layouts (python-pptx)

python-pptx exposes the `slide_layouts` from the default or template theme. Common indices in the default theme:

| Index | Layout |
| --- | --- |
| 0 | Title Slide (title + subtitle) |
| 1 | Title and Content |
| 2 | Section Header |
| 3 | Two Content |
| 5 | Title Only |
| 6 | Blank |

With a custom template, layouts differ. List them to confirm: `for i, l in enumerate(prs.slide_layouts): print(i, l.name)`.

## Common bugs

- **Text overflows the placeholder.** python-pptx won't shrink-fit automatically. Either reduce content, reduce font size, or use `text_frame.word_wrap = True` and enable auto-fit via `MSO_AUTO_SIZE.TEXT_TO_SHAPE_AUTO_SIZE` in the shape's text frame.
- **Images overflow the slide.** Compute the insertion size: standard slide is 10"×7.5" (`Inches(10)`, `Inches(7.5)`). Clamp image dimensions.
- **Fonts missing on the projector machine.** Stick to system fonts (Calibri, Arial). Exotic fonts substitute at presentation time.
- **Pandoc output ugly and uncustomizable.** Switch to python-pptx.
- **Corrupted file on open.** Almost always a mismatched placeholder type. Check `slide.placeholders.keys()` before assigning to indices that don't exist in that layout.

## Adding images, tables, speaker notes

```python
slide.shapes.add_picture('chart.png', Inches(1), Inches(2), width=Inches(8))

rows, cols = 3, 4
table = slide.shapes.add_table(rows, cols, Inches(1), Inches(1.5), Inches(8), Inches(4)).table
table.cell(0, 0).text = "Header"

notes_slide = slide.notes_slide
notes_slide.notes_text_frame.text = "Speaker notes for this slide."
```

## Stopping criterion

Every slide has a title, no content clipped, layout consistent across slides, image/table placement correct. Ship it.
