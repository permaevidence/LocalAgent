# PPTX Skill

Presentations are layout-heavy and slide-bounded. Unlike PDFs, content on a slide can't flow to the next page — if text overflows, it just disappears off the bottom. Verification must confirm content fits each slide.

## Renderer choice

**Primary: python-pptx.** The only serious tool for generating .pptx programmatically. Full control over layouts, shapes, text frames, images, tables, speaker notes.
```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation()  # or Presentation('template.pptx') to inherit theme
slide = prs.slides.add_slide(prs.slide_layouts[1])  # Title + Content layout
slide.shapes.title.text = "Slide Title"
slide.placeholders[1].text = "Body bullet 1"
prs.save('output.pptx')
```

**Secondary: pandoc markdown → pptx.** Quick for content-only decks without custom styling. Limited control.
```bash
pandoc deck.md -o output.pptx --reference-doc=template.pptx
```

## Workflow

1. **Ask for a template if branded.** Corporate decks have brand colors, fonts, a logo on every slide. Use `Presentation('template.pptx')` to inherit. Don't invent brand identity.
2. **Sketch the structure first.** What slides? What goes on each? Block out content before writing code. 10 slides × 1 title + 3-5 bullets is a common shape.
3. **Generate.**
4. **Verify visually.** PPTX isn't inline-viewable, so convert to PDF and inspect:
   ```bash
   libreoffice --headless --convert-to pdf output.pptx
   ```
   Then `read_file output.pdf` — rendered slides come back as inline multimodal content.
5. **Sample EVERY slide, not just the first.** Overflow bugs, missing titles, broken layouts typically show up on specific slides, not systematically. Verify each slide:
   - Has a visible title
   - No text clipped at the bottom (overflow)
   - Images don't cover text or overflow the slide
   - Body text legible at projector scale — 18-24pt body, 28-40pt titles
   - No empty slides (generated by accident)
   - Consistent visual style (no random color or font jumps)
6. **Fix and re-render.** Cap at 3 iterations.

### Pre-render sanity check

Before the full render-convert-inspect cycle, a quick python-pptx walk catches mechanical bugs:
```python
from pptx import Presentation
prs = Presentation('output.pptx')
print(f"Slides: {len(prs.slides)}")
for i, slide in enumerate(prs.slides, 1):
    title = slide.shapes.title.text if slide.shapes.title else "(no title)"
    print(f"  {i}. {title}")
```
Catches "I wanted 10 slides, I have 8" before you waste a render cycle.

## Layout discipline

Three rules that catch 90% of bad decks:

1. **One idea per slide.** More than 6 bullets = split into two slides.
2. **Titles are sentences, not labels.** "Q3 revenue up 23%" beats "Q3 Revenue."
3. **Leave whitespace.** Margins matter — don't fill every pixel.

## Using layouts (python-pptx)

Default theme layout indices:

| Index | Layout |
| --- | --- |
| 0 | Title Slide (title + subtitle) |
| 1 | Title and Content |
| 2 | Section Header |
| 3 | Two Content |
| 5 | Title Only |
| 6 | Blank |

Custom templates differ. List them: `for i, l in enumerate(prs.slide_layouts): print(i, l.name)`.

## Adding images, tables, speaker notes

```python
slide.shapes.add_picture('chart.png', Inches(1), Inches(2), width=Inches(8))

table = slide.shapes.add_table(3, 4, Inches(1), Inches(1.5), Inches(8), Inches(4)).table
table.cell(0, 0).text = "Header"

slide.notes_slide.notes_text_frame.text = "Speaker notes for this slide."
```

## Common bugs

- **Text overflows the placeholder.** python-pptx won't shrink-fit automatically. Reduce content, reduce font size, or enable auto-fit via `MSO_AUTO_SIZE.TEXT_TO_SHAPE_AUTO_SIZE` on the shape's text frame.
- **Images overflow the slide.** Standard slide is 10"×7.5" (`Inches(10)`, `Inches(7.5)`). Clamp image dimensions.
- **Fonts missing on the projector machine.** Stick to system fonts (Calibri, Arial) unless the user specifies.
- **Pandoc output ugly and uncustomizable.** Switch to python-pptx.
- **Corrupted file on open.** Almost always a mismatched placeholder type. Check `slide.placeholders.keys()` before assigning to indices that don't exist in the layout.

## Stopping criterion

Every slide has a title, no content clipped, layout consistent across slides, image/table placement correct. Ship it.
